1. Папка "common_config" содержит в себе конфиг nginx-proxy. Nginx-proxy будет обрабатывать запрос в браузере и раскидывать на соответствующие сервера, которые могут быть уже как на nginx так и на apache. Находясь в этой папке необходимо запустить команды  "docker network create nginx-proxy" и "docker-compose up -d"
2. В папке "server_configs" хранятся различные конфиги серверов. Как управлять конфигами. В корне "server_configs" есть файл "docker-compose.yml". В нем необходимо менять следующие настройки:
- в блоке php-apache задаем свойства SITENAME и VIRTUAL_HOST со значением название сайта (пример есть в конфиге)
- чтобы поменять версию php необходимо в блоке "php-apache" у свойства "dockerfile" указать папку "dockerfile-55" для версии php 5.5
- чтобы поменять версию mysql необходимо в блоке "mysql" у свойства "dockerfile" указать папку "dockerfile-56" для версии mysql 5.6
3. В папке "php-apache" в файлах "hosts" и "virtual_host_site.conf" необходимо также вписать название сайта
4. Команда для поднятия сервера: "docker-compose build && docker-compose up -d"
5. Сейчас созданы конфиги для php 5.4, 5.5, 5.6 и для mysql 5.5, 5.6, 5.7. Конфигурации проверены на битриксе. Установка проходит в штатном режиме. Конфигурации настроены под utf-8. ВНИМАНИЕ! Не всегда создается БД по умолчанию (site), если бд не создалась, то можно её создать вручную, либо при установке битрикса задать создание новой БД
